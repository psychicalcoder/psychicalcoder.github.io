演算法週記 單源最短路徑之Bellman-Ford演算法

2017-08-02

最短路徑

什麼是最短路徑呢?
首先最短路徑要有起點(SSS)跟終點(TTT)
從SSS走到TTT經過的邊權總和最短的一條路徑就稱為”最短路徑”(Shortest Path)
而這個邊權和就是”最短距離”

最短路徑有好多種演算法
今天我們來看看Bellman-Ford演算法

Bellman-Ford Algorithm

這是個相當常用的演算法
他除了可以找出最短路徑之外
還能找出負環

將從起點SSS到終點YYY的最短路徑設為d[y]d[y]d[y]
那麼怎麼找出d[y]d[y]d[y]呢?
首先呢，SSS到YYY的路徑一定是從某個頂點XXX由邊e(x,y)e(x,y)e(x,y)延伸過去的
那麼假設SSS到XXX的最短路徑都已經確定了
則下列的式子一定會成立
d[y]=min{d[x]+cost(x,y)∣e(x,y)∈E}d[y] = min{d[x]+cost(x,y)|e(x,y) \in E}d[y]=min{d[x]+cost(x,y)∣e(x,y)∈E}
其中cost(x,y)cost(x,y)cost(x,y)代表邊e(x,y)e(x,y)e(x,y)的權重

那麼要怎麼確保所有的XXX的最短路都已經被確定了呢?

其實不需要這麼做
因為最短距離只會變小不會變大
所以只要在能讓最短距離變小的時候更新
到最後就會是真的最短距離了喔!

以下就是Bellman-Ford Algorithm的核心
1.將d[]d[]d[]初始化成INF
2.將d[s]d[s]d[s]設為000
3.掃過所有的邊e(x,y)e(x,y)e(x,y)
4.對於每個邊e(x,y)e(x,y)e(x,y)的x,yx,yx,y如果d[x]d[x]d[x]不是初始值且d[y]>d[x]+cost(x,y)d[y] > d[x]+cost(x,y)d[y]>d[x]+cost(x,y)成立
5.則更新d[y]=d[x]+cost(x,y)d[y] = d[x]+cost(x,y)d[y]=d[x]+cost(x,y)
6.如果掃過一次所有邊有任何更新則跳回步驟3否則繼續步驟7
7.結束演算法

接著我們來分析一下他的複雜度吧!

考慮一下沒有負環的情況(路徑中有負環的話，最短路徑不存在，迴圈也跑不完)
在沒有負環的情況下
最短路徑不會重複通過同個頂點
那麼今天有條很長的最短路徑
對於路徑上的每個點來說
這條路徑一定是最短路

考慮某個已經被確定的頂點AAA的連外頂點BBB
BBB的最短路徑上的前一個點是AAA的話
則最多在AAA被確定後的下個迴圈
一定也會被確定(也可能在同個迴圈就被確定)
也就是說迴圈最多會跑VVV次(V−1V-1V−1個待確認的頂點+一次沒有用的迴圈)
所以說時間複雜度是O(VE)O(VE)O(VE)鄰接矩陣的話則是O(V3)O(V^3)O(V​3​​)

利用最多會跑VVV次迴圈的這個特性
我們可以統計迴圈的次數
如果超過V−1V-1V−1次還在更新的話
就能確定這張圖有負環
